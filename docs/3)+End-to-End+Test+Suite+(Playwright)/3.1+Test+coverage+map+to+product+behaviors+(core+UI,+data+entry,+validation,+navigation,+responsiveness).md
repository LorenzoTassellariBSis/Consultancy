# End-to-End Test Suite (Playwright)

## 3.1 Test Coverage Map to Product Behaviors

This section maps each behavior verified in `time-tracker.spec.js` to the corresponding product requirement and DOM contract. All tests run against the static Time Tracker application served at `http://localhost:8080/index.html` .

### Core UI

- **Page title and header**

Verifies that the browser title is “Time Tracker”, the main heading `<h1>` reads “Time Tracker”, and the subtitle `.subtitle` reads “Track your work days this month” .

- **Current month badge**

Ensures the month badge `#currentMonth` is visible and contains the current year .

- **Initial total display**

Confirms the total days counter `#totalDays` starts at `0` .

- **Week-card generation**

Checks that `.week-card` elements are generated in a count between 4 and 6 for the current month .

- **Week-card structure**

Validates within the first week card:

- Title `.week-title` is visible and reads “Week 1”
- Days display `.week-days` is present
- Input field `input[type="number"]` is present
- Save button `button` is visible and labeled “Save” .

### Data Entry & Persistence

- **Entering and saving days worked**

Filling `#week1-input` (e.g. “5”) and clicking Save updates `#week1-display` to the entered value .

- **Updating existing entries**

Overwriting an existing value (e.g. from “3” to “7”) updates both the week display and the aggregate total accordingly .

- **Totals aggregation**

Saving days across multiple weeks (e.g. 3 in week 1 and 4 in week 2) updates `#totalDays` to the sum (7) .

- **Multiple-week looped entries**

Iterating over an array of week/day pairs correctly accumulates and displays the total .

- **Input retention for editing**

After saving, the input retains its value to allow further edits, as verified by comparing input value before and after Save .

### Validation

- **Range enforcement (0–7 days)**

Entering a value outside 0–7 (e.g. “8”) triggers a browser alert with message “Please enter a value between 0 and 7 days” .

- **Zero-days handling**

Entering “0” and saving displays “0” in the week display without error .

### Month Navigation

- **Navigation controls**

The Previous (`#prevMonth`) and Next (`#nextMonth`) buttons are visible and clickable .

- **Badge and week-cards update**

Clicking Next or Previous updates the `#currentMonth` badge and regenerates `.week-card` elements (count remains 4–6) .

- **Per-month data isolation**

Data saved in one month resets when navigating to a different month, then is restored when navigating back, ensuring separate storage per month .

### Responsiveness

- **Desktop viewport**

At 1200 × 800, the container `.weeks-grid` is visible .

- **Mobile viewport**

At 375 × 667, `.weeks-grid` and individual `.week-card` elements remain visible and usable .

### Animation Feedback

- **Success animation on save**

Saving a valid number of days applies the `.success-animation` class to the week display (e.g. `#week1-display`) and to the total counter, providing visual feedback .
